# AGENTS.md

## Cursor Cloud specific instructions

### Overview

C55 Tracker is a local-first predictive-maintenance dashboard (Next.js 16 + Python data pipeline). It works entirely with local JSON files by default — Supabase/Postgres is opt-in and only activates when `SUPABASE_SERVICE_ROLE_KEY` is set.

### Standard commands

All commands are documented in `README.md`. Quick reference:

| Action | Command |
|--------|---------|
| Dev server | `npm run dev` (port 3000) |
| Lint | `npm run lint` |
| Tests | `npm test` |
| Python pipeline | `python data_pipeline.py` |

### Non-obvious caveats

- **No Supabase required for local dev.** The app falls back to `data/lifecycles.json` (local-JSON backend) when `SUPABASE_SERVICE_ROLE_KEY` is unset. All API routes, the dashboard, and tests work without any database.
- **Python deps install to user site.** `pip install -r requirements.txt` installs to `~/.local` since the system site-packages is read-only. Python scripts (`data_pipeline.py`, `scripts/*.py`) are optional for dashboard dev — they process raw Excel/CSV source files that are gitignored.
- **`data/` directory is gitignored and created on first API write.** The lifecycle store auto-creates `data/lifecycles.json` on first mutation (part replacement, maintenance log, etc.).
- **`public/pipeline.json` and `public/lifecycles.json` are committed.** These seed files bootstrap the dashboard with sample data. They are regenerated by `data_pipeline.py` and `scripts/import_tracker.py` respectively, but the committed versions are sufficient for development.
- **ESLint 9 flat config** is used (`eslint.config.mjs`). The single existing warning (`_opts` unused in `vantagepoint-adapter.ts`) is pre-existing.
- **Tests use `node:test` (built-in)** with `tsx` for TypeScript support: `node --import tsx --test lib/__tests__/*.test.ts`.
- **Next.js 16 with React 19** — uses the App Router exclusively (no `pages/` directory).
